1.Commit Message 应该写什么

````
    业界最通用的是 Conventional Commits 规范，格式如下：
    <type>(<scope>): <subject>

    <body>

    <footer>
````

2.常用 Type 类型：
````

    类型	        含义	                示例
    feat        新功能	            feat(auth): add JWT token refresh mechanism
    fix         修复                 Bug	fix(api): resolve null pointer in user query
    docs	    文档更新	            docs(readme): update installation steps
    style	    代码格式（不影响逻辑）	style: remove trailing spaces
    refactor	重构（非 feat/fix）	refactor(utils): simplify date parsing logic
    perf	    性能优化	            perf(db): add index to user_id column
    test	    测试相关	            test(auth): add unit tests for login flow
    chore	    构建/工具/杂项	    chore(deps): bump lodash to 4.17.21
    revert	    回滚	revert:         feat(auth): add JWT token refresh mechanism

````
3.编写原则：

    1.Subject（标题行）

    用祈使句（Add/Fix/Update，而非 Added/Fixed）
    首字母小写，结尾不加句号
    不超过 50 个字符，清晰说明"做了什么"

    2.Body（正文，可选）
    
    详细说明为什么做这改动
    解释实现思路或 breaking changes

    3.Footer（页脚，可选）

    关联 Issue：Closes #123, Fixes #456
    Breaking Change 标记：BREAKING CHANGE: ...
4.示例：
    
    feat(file-upload): support drag-and-drop for multiple files

        - Add dropzone component with visual feedback
        - Handle edge cases for large files (>100MB)
        - Update error messages for better UX

    Closes #789
